# Agent Reference

Situational guidance for coding agents. Open these files only when relevant to the task.

Rules that must apply to every task belong in `../AGENTS.md`.

## Files

- `stack-decisions.md`: current project stack and architectural choices.
- `security-guardrails.md`: security review checklist.
- `data-access.md`: data ownership, authorization, and persistence guidance.
- `threat-modeling.md`: lightweight threat modeling guidance.
- `dependency-policy.md`: criteria for adding, updating, or removing dependencies.
- `skill-preferences.md`: user preferences for creating or updating agent skills.

## Maintenance

- Keep each file concise and skimmable.
- Prefer checklists over prose.
- Remove stale guidance when decisions change.
- Avoid duplicating `AGENTS.md`.
